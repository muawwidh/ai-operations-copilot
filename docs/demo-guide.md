# Demo Guide

This document explains how to demonstrate AI Operations Copilot.

The demo should show the system as a full-stack AI operations assistant with analytics, AI chat, document intelligence, reporting, and Dockerized deployment.

---

## 1. Demo Objective

The demo should show that AI Operations Copilot can:

- Display operational KPIs
- Analyze shipment delays
- Summarize support ticket and SLA risks
- Identify active customers
- Answer natural language business questions
- Upload and understand PDF documents
- Generate management reports
- Run as a dockerized full-stack application

---

## 2. Start the System

From the repository root:

```bash
docker compose up --build
```

Or background mode:

```bash
docker compose up --build -d
```

---

## 3. Open the Application

Frontend:

```text
http://127.0.0.1:8501
```

Backend Swagger:

```text
http://127.0.0.1:8000/docs
```

Backend Health Check:

```text
http://127.0.0.1:8000/api/health
```

## 4. Demo Flow Summary

Recommended demo order:

```text
1. Dashboard
2. AI Chat
3. Reports
4. Documents
5. Swagger API
6. Docker explanation
```

This order shows business value first, then technical architecture and implementation depth.

## 5. Dashboard Demo

Open the Streamlit frontend:

```text
http://127.0.0.1:8501
```

Go to:

```text
Dashboard
```

Show the following metrics:

- Customers
- Shipments
- Delayed shipments
- Delay rate
- Support tickets
- Open tickets
- High-priority tickets
- SLA breach rate

Then show:

- Shipment delay reasons chart
- Ticket categories chart
- Most active customers table

### Suggested Explanation

```text
This dashboard shows operational analytics from PostgreSQL. The backend calculates shipment delays, ticket status, SLA risk, and customer activity. The frontend displays the results through Streamlit.
```

---

## 6. AI Chat Demo

Go to:

```text
AI Chat
```

Ask:

```text
Why are shipments delayed this week?
```

Expected result:

- AI uses shipment delay tool
- Answer explains delay reasons
- Tools used are shown below the answer

Then ask:

```text
Summarize the support ticket situation and SLA risks.
```

Expected result:

- AI uses ticket summary tool
- Answer explains SLA breach risk and support workload

Then ask:

```text
Which customers are most active?
```

Expected result:

- AI uses customer activity tool
- Answer lists active customers and business meaning

### Suggested Explanation

```text
The chat assistant uses agent tool calling. The model selects an approved backend tool, the backend executes the tool safely, and the final response is generated in business language.
```

---

## 7. Report Demo

Go to:

```text
Reports
```

Enter report type:

```text
Weekly Operations Report
```

Generated by:

```text
operations_manager
```

Click:

```text
Generate Report
```

Expected result:

- AI generates a structured management report
- Report is saved in PostgreSQL
- Report appears in report history

Then show report history and open the saved report.

### Suggested Explanation

```text
The system generates management reports using analytics context from the database. Each report is saved, making it part of the system history rather than a temporary AI response.
```

---

## 8. Document Intelligence Demo

Go to:

```text
Documents
```

Upload a text-based PDF containing readable business, policy, process, or operational content.

After upload, show:

- File name
- Processed status
- Number of chunks created

Ask:

```text
Summarize the main rules in this document.
```

Then ask:

```text
What does this document say about escalation?
```

Expected result:

- System retrieves relevant chunks from ChromaDB
- AI answers using retrieved document context
- Retrieved chunks can be viewed in the debug section

### Suggested Explanation

```text
This demonstrates the document intelligence workflow: PDF ingestion, text extraction, chunking, embedding generation, semantic retrieval, and AI-based answer generation using retrieved context.
```

---

## 9. Swagger API Demo

Open:

```text
http://127.0.0.1:8000/docs
```

Show API groups:

- Health
- Analytics
- AI Chat
- Documents
- Reports

Test:

```text
GET /api/health
```

Then test:

```text
GET /api/analytics/operations-summary
```

Then test:

```text
POST /api/chat/ask
```

Example body:

```json
{
  "question": "Give me a high-level operations performance overview."
}
```

### Suggested Explanation

```text
The backend is exposed through FastAPI REST endpoints. The Streamlit frontend is one client, but other systems can also integrate with the backend through the API.
```
